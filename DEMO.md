## 🎯 このシステムについて

**「紙の出席簿、まだ使ってますか？」**

本システムは、QRコードを使った次世代の出席管理システムです。
教員がQRコードを表示し、学生がスマホでスキャンするだけで出席完了。
入室・退室の記録、出席申請、レポート出力まで、すべてをWebで完結します。

### 開発の背景
- 紙の出席簿は集計が大変、紛失リスクもある
- 代返（なりすまし出席）を防ぎたい
- リアルタイムで出席状況を把握したい

### 解決策
✅ **QRコード＋位置情報**で本人確認  
✅ **入室/退室の2回スキャン**で滞在時間を記録  
✅ **自動集計＋レポート出力**で教員の負担軽減

---

## ✨ 主な機能

| 機能 | 説明 |
|------|------|
| 🔲 **QRコード出席** | 教員が発行したQRを学生がスキャンして出席登録 |
| 📍 **位置情報認証** | 校内（半径50m）でのみ出席可能 |
| 🚪 **入室/退室記録** | 2回スキャンで滞在時間・早退を自動判定 |
| 📝 **出席申請** | 遅刻・欠席・公欠の理由を申請→教員が承認 |
| 📊 **レポート** | 週次/日次の出席率推移、PDF/CSV出力 |
| 🔔 **通知** | 出席状況の変化をリアルタイム通知 |
| 🔒 **監査ログ** | すべての操作を記録、不正を防止 |

## 💡 ここがポイント！

### 🛡️ 不正防止の工夫
- **位置情報認証**: OIC校内（半径50m）でのみ出席可能
- **時間制限付きQR**: QRコードは定期的に更新
- **監査ログ**: すべての操作を記録

### 📱 使いやすさへのこだわり
- **スマホ最適化**: タップしやすいボタンサイズ
- **リアルタイム更新**: ページ遷移なしで即座に反映
- **オフライン対応**: 電波が弱くてもスキャン可能

### 🔧 技術的な特徴
- **Ruby on Rails 8** + **Hotwire** でリアルタイムUI
- **PostgreSQL** で堅牢なデータ管理
- **PWA対応** でアプリのような使い心地

---

## 🖥️ 動作環境

- **スマートフォン**: カメラ付きのスマホ（iOS/Android）
- **PC**: Chrome / Safari / Edge など主要ブラウザ
- **ネットワーク**: インターネット接続が必要
