# デモ動画（3〜5分）構成案

## 目的
- 学生 / 先生 / 管理者の3ロールを短時間で体験できる導線を見せる
- 主要機能（QR出席・申請・通知・レポート・管理）を一通り確認できる流れにする

## 事前準備（撮影前に済ませる）
- テスト用アカウントを用意（学生 / 先生 / 管理者）
- 代表的なデータを用意
  - 学生: 1名分の出席履歴・出席申請履歴
  - 先生: 1クラス分の名簿・出席データ・申請
  - 管理者: 複数クラスの集計データ
- 画面サイズと解像度を固定（例: 1280x720）

## 0. オープニング（10〜20秒）
- 画面: タイトル or ログイン画面
- ナレーション例:
  - 「このデモでは、学生・先生・管理者それぞれの画面を通して、QR出席管理がどうスムーズに回るかを紹介します。」

## 1. 学生ロール（60〜90秒）
- 1-1. ログイン
  - 画面: 学生でログイン
  - 伝えること: 「学生は自分の出席状況をすぐ確認」
- 1-2. QRスキャンで出席登録
  - 画面: QRスキャン → 出席完了
  - 伝えること: 「操作は最短で完了」
- 1-3. 出席履歴の確認
  - 画面: 出席履歴カレンダー → 詳細
  - 伝えること: 「出席状況が一目で分かる」
- 1-4. 出席申請
  - 画面: 欠席/遅刻/公欠の申請
  - 伝えること: 「申請も簡単に送れる」

## 2. 先生ロール（60〜90秒）
- 2-1. ログイン
  - 画面: 先生でログイン
  - 伝えること: 「クラス全体の状況を俯瞰」
- 2-2. QRコード発行
  - 画面: QR発行 → 表示
  - 伝えること: 「QRを発行して出席を取る」
- 2-3. 出席確認
  - 画面: 授業ごとの出席一覧
  - 伝えること: 「未出席の学生がすぐ把握できる」
- 2-4. 申請承認
  - 画面: 申請一覧 → 承認/却下
  - 伝えること: 「申請の処理も簡単」

## 3. 管理者ロール（60〜90秒）
- 3-1. ログイン
  - 画面: 管理者でログイン
  - 伝えること: 「全体管理のダッシュボード」
- 3-2. クラス別出席集計
  - 画面: クラス別の出席率
  - 伝えること: 「傾向をすぐ把握」
- 3-3. ユーザー管理
  - 画面: 学生/先生の登録・編集
  - 伝えること: 「管理作業もここで完結」

## 4. クロージング（10〜20秒）
- 画面: サマリ or トップ画面
- ナレーション例:
  - 「学生・先生・管理者のそれぞれの業務が、最小の操作でつながることが分かります。」

## 目安の時間配分
- オープニング: 0:10〜0:20
- 学生: 1:00〜1:30
- 先生: 1:00〜1:30
- 管理者: 1:00〜1:30
- クロージング: 0:10〜0:20
