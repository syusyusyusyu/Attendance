# アプリの良いところと改善ポイント

## 概要
- 対象: 学校/専門学校向けの出席管理に特化したWebアプリ
- 目的: 出席登録の高速化、誤記録の削減、監査性の確保
- 方式: QRセッション + 署名トークンで入室/退室を記録
- ロール: 学生/教員/管理者の3ロール運用
- 学生体験: QRスキャン→出席、履歴/通知/申請が1画面
- 教員体験: 出席確認/修正/承認/確定まで一気通貫
- 管理者体験: ユーザー/権限/監査/承認/端末/APIを統合管理
- 中核機能: 出席ポリシー、授業回、監査ログ、レポート、通知
- 不正対策: QRローテーション、IP/端末制限、レート制限、異常検知

## 良いところ
- QRセッションと署名トークンで改ざん耐性のある出席登録
- 入室/退室の2回スキャンで滞在時間を記録し早退判定まで自動化
- クラス単位で出席ポリシーと不正検知しきい値を調整可能
- 監査ログの検索・CSV出力・条件保存で運用監査に強い
- 申請承認/操作承認のワークフローで変更履歴が追える
- 管理者向けにユーザー/権限/端末/APIを一元管理
- 期末レポート(PDF/CSV)と可視化グラフで説得力を強化
- 通知/履歴/申請の導線が短く、学生の自己管理を支援
- CSV入出力がBOM付きでExcel互換に配慮
- モバイルでのQRスキャン/手入力フォールバックが安定

## 改善できるところ(次の一歩)
- 監査ログの全文検索をDBネイティブに最適化
- 高負荷時の非同期化(通知/集計/レポート生成のジョブ化)
- 外部通知(LINE/Push)の実運用連携と再送制御
- 監査ログの改ざん耐性(署名チェーン)の強化
- セキュリティ強化(MFA、端末バインド、IPレンジ管理)
- 負荷テスト/セキュリティテスト/DR手順の自動化

## 完成版の機能一覧(業務運用 + 作品展)
### 学生
- QRスキャン/手入力フォールバック、入室/退室の可視化
- 出席履歴、申請(欠席/遅刻/公欠)、申請結果の通知
- 履修クラス、当日ステータス、次回授業の確認
- 端末登録、通知設定、プロフィール編集

### 教員
- QR発行/失効/再発行、授業回の確定/解除
- 出席確認/一括修正/申請承認、CSV入出力
- 出席ポリシー設定(遅刻/締切/早退/警告閾値)
- 特別日程(補講/休講)管理、出席ログの検索/CSV
- レポート(週次/日次/期末)と要注意者抽出

### 管理者
- ユーザー/ロール/権限の管理と権限監査
- 操作申請の承認、監査ログの条件保存/集計
- 端末承認、APIキー管理
- データ保持/バックアップ/復旧手順の運用管理

### 不正対策/セキュリティ
- QRローテーション、IP/端末制限、レート制限
- 不正検知アラート(失敗多発/異常IP/QR共有)
- 監査ログの理由必須化と改ざん耐性

### 分析/可視化
- 出席率推移、遅刻/早退理由分布、学生別推移
- 期末レポート(PDF/CSV)の根拠提示

### 運用/品質
- 監視/アラート、SLO運用、負荷/セキュリティ試験
- 教員/学生向けの操作ガイドとFAQ

## プレゼン用まとめ
### 1分スクリプト
- 「出席を速く、正確に、監査可能にする」ことを目標に設計しました。
- QRセッション+署名トークンで入室/退室を記録し、遅刻/早退を自動判定します。
- 教員は出席確認/修正/承認/確定を1画面で完結できます。
- 管理者は権限・監査・承認・端末・APIを統合管理できます。
- 不正対策と監査ログまで備え、実運用を想定した完成度です。

### 5枚スライド構成(例)
1. 目的: 出席の正確性と監査性を両立
2. 仕組み: QRセッション/署名トークン/入退室記録
3. ユーザー体験: 学生・教員・管理者の導線
4. 運用: 承認フロー/監査ログ/不正検知
5. 成果: レポートと拡張性

### デモの流れ(90秒)
1. 教員ログイン → QR発行
2. 学生スキャン → 出席登録
3. 再スキャン → 退室登録(早退判定)
4. 教員の承認/確定
5. レポート出力
