# アプリの良いところと改善ポイント

## 概要
- 対象: 学校/専門学校向けの出席管理に特化したWebアプリ
- 目的: 出席登録の高速化、誤記録の削減、監査性の確保
- 方式: QRセッション + 署名トークンによる入室/退室記録
- 役割: 学生/教員/管理者の3ロールで運用する設計
- 学生体験: QRスキャンで出席登録、履歴/通知/申請が1画面で完結
- 教員体験: 出席確認/修正、申請承認、確定、監査ログの一気通貫
- 管理者体験: 教員機能の全権 + 確定解除/強制修正
- 中核機能: 出席ポリシー、授業回、監査ログ、レポート、通知
- 早退判定: 入室/退室の滞在時間から自動判定
- 出席申請: 欠席/遅刻/公欠の申請と承認フロー
- 授業回確定: 締切後に自動欠席確定し、運用を固定化
- 監査性: 変更理由必須 + 変更履歴/CSVで可視化
- 不正対策: QRローテーション、IP/端末制限、レート制限
- レポート: 週次/日次推移、要注意者抽出、期末PDF/CSV
- クラス運用: クラス管理/名簿CSV/休講補講/履修管理
- リアルタイム: Hotwireで出席反映が即時
- モバイル対応: カメラ非対応端末は手入力にフォールバック
- 展示向け: デモデータ/デモシナリオ/QAが整備済み
- 文書整備: 仕様書/運用ガイド/QA/レビューが揃っている
- デプロイ: Render向けにビルド/運用コマンドを用意

## 良いところ
- QRセッション + 署名トークンで出席登録の信頼性が高い
- 入室/退室を2回スキャンで記録し、早退判定まで実装されている
- 出席ポリシーがクラス単位で設定でき、遅刻/締切判定が自動化されている
- 監査ログ(出席変更ログ/スキャンログ)があり、運用に強い
- 出席申請(欠席/遅刻/公欠)の承認フローがあり実務に近い
- 授業回の確定/解除と自動欠席確定があり運用フローが明確
- QRローテーションやIP/端末制限など不正対策の設計が進んでいる
- レポート(週次/日次推移、要注意者抽出、期末PDF/CSV)が揃っている
- 通知機能があり、更新の見落としを防げる
- クラス管理/名簿CSV/特別日程で学校運用に対応できる
- Hotwireでリアルタイム性があり、出席反映が速い
- 画面が教員/学生で明確に分かれていて導線が分かりやすい
- シード/デモデータとデモシナリオが用意されており展示に強い
- 仕様書/QA/運用ガイドが整備され、引き継ぎ性が高い
- CSV出力がBOM付きでExcel互換性を意識している
- 監査ログに理由必須があり、変更理由の可視化ができる
- 運用を見越したCron実行タスクがある

## 改善できるところ
- 管理者専用の管理画面(ユーザー/権限/監査の統合)を追加する
- 権限の粒度をロール/権限テーブルで細分化する
- 監査ログの検索性(全文検索/条件保存/ダッシュボード集計)を強化する
- 期末レポートのPDFをテーブル/集計グラフ付きにする
- レポート画面にグラフ(線/棒)を追加し視認性を上げる
- 出席申請の承認UIを一括処理/ショートカット対応にする
- 学生側の履修/スキャン/履歴の導線をさらに短くする
- 学生ログインの初回案内(チュートリアル/FAQ/動画)を追加する
- QRスキャンの失敗理由をユーザー向けにもう少し具体化する
- カメラ権限/対応ブラウザの案内をスキャン画面に常時表示する
- モバイルUIの固定ヘッダー/サイドバー重なりを完全に解消する
- 大量同時アクセス時の最適化(キャッシュ/インデックス/非同期処理)を強化する
- QRスキャンのレート制限を教室単位/学生単位で調整可能にする
- 学務連携向けにAPIエンドポイントと認可方式を整備する
- 出席データの修正ワークフロー(承認フロー)を追加する
- 不正検知アラートの閾値/ルールを管理UIで編集可能にする
- 端末ごとのデバイス登録(公認端末)を追加する
- SSO(学校アカウント)連携を追加する
- 教員/学生の一覧に検索・フィルタ・保存条件を追加する
- 出席履歴にエクスポート(個人CSV/PDF)を追加する
- 通知の配信先(メール/LINE/Push)を選択できるようにする
- 重要操作(確定/解除/CSV反映)に承認/確認フローを追加する
- 利用規約/プライバシー/データ保持ポリシーを整備する
- 本番運用での障害対応(バックアップ/復旧/監視)を手順化する
- テストの拡充(システムテスト/負荷/セキュリティ)を追加する
- UIの色・余白・タイポグラフィを統一し、完成度を上げる

## 追加で伸ばせる方向性(発表で強くなるポイント)
- デモ用の「成果ストーリー」(課題→解決→効果)を画面で可視化する
- 出席データから「学習支援アラート」を出すなど教育価値を示す
- 教員の業務削減時間や出席精度の改善率を数値で示す
- QR以外の手段(GPS/ビーコン/IC)との拡張性を見せる
- 見た目だけでなく「運用/監査/不正対策」が実務準拠である点を強調する

## プレゼン用まとめ

### 1分版(口頭スクリプト)
- 私たちは「出席を速く、正確に、監査可能にする」ことを目的に出席管理を作りました。
- QRセッション+署名トークンで入室/退室を記録し、遅刻・早退も自動判定します。
- 教員は出席確認/修正/確定を1画面で完結、変更理由は必須で監査ログに残ります。
- 学生はQRスキャンで登録し、履歴・通知・申請まで1つの導線で完了します。
- 不正対策/レポート/デモデータまで揃えて、実運用に耐える完成度を目指しました。

### 3分版(スライド構成案)
1. 課題: 手作業の出席管理は遅く、ミスと不正に弱い
2. 解決: QRセッション+署名トークンで入室/退室を記録
3. 体験: 学生はスキャン、教員は承認/確定、管理者は解除まで統制
4. 実務性: 監査ログ/通知/レポート/ポリシーで運用が回る
5. 効果: 記録速度向上、変更履歴可視化、教育支援の土台

### 5枚スライドの内容(例)
1. タイトル/目的: 「出席を速く、正確に、監査可能に」
2. ユーザーフロー: 学生/教員/管理者の導線を1枚で
3. コア機能: QRセッション、遅刻/早退判定、申請承認
4. 運用/監査: 変更理由必須、スキャンログ、確定フロー
5. レポート/今後: 週次/日次推移、要注意者、拡張余地

### デモの流れ(90秒)
1. 教員でログイン → QR発行
2. 学生でスキャン → 出席登録
3. 再スキャン → 退室登録(早退判定の表示)
4. 教員画面でリアルタイム反映
5. 申請承認 → 出席確定 → レポート出力

### 強みの一言キャッチ
- 「QRの速さ」と「監査の強さ」を両立した出席管理
- 操作ミスより運用フローを先に設計した現場志向の仕組み

### 想定質問に対する即答
- Q: 不正対策は? A: QRローテーション、IP/端末制限、レート制限とログ監査。
- Q: 早退判定は? A: 入退室の滞在時間と最低出席率で自動判定。
- Q: 実運用できる? A: 変更理由必須+確定フロー+通知/レポートで運用可能。
